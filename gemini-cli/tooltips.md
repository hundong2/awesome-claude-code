# Gemini CLI 유용한 툴팁

Gemini CLI의 생산성을 높여주는 몇 가지 유용한 팁과 트릭입니다.

## 시작하기 & 기본 명령어

*   **설치**: `npm install -g @google/gemini-cli` 명령어로 Gemini CLI를 전역으로 설치할 수 있습니다. 또는 `npx @google/gemini-cli`를 사용하여 설치 없이 실행할 수도 있습니다.
*   **대화형 모드**: `gemini`를 실행하여 대화형 세션을 시작합니다.
*   **단일 응답 모드**: 단일 응답을 얻으려면 `-p` 플래그와 함께 프롬프트를 입력합니다. (예: `gemini -p "your prompt"`)
*   **도움말**: 언제든지 `/help` 또는 `/?`를 입력하여 사용 가능한 명령어와 단축키 목록을 볼 수 있습니다. `Ctrl+L`로 화면을 지울 수도 있습니다.

## 컨텍스트 및 메모리 관리

*   **`GEMINI.md`로 컨텍스트 유지**: 프로젝트에 일관된 지침을 제공하려면 프로젝트의 루트 디렉토리에 `.gemini` 폴더를 만들고 `GEMINI.md` 파일을 추가하세요. CLI는 모든 프롬프트에 대해 이 파일의 내용을 자동으로 컨텍스트로 로드합니다. `/init` 명령어를 사용하여 `GEMINI.md` 파일을 자동으로 생성할 수 있습니다.
*   **메모리 보기 및 새로고침**: `/memory show`를 사용하여 `GEMINI.md` 파일에서 로드된 현재 컨텍스트를 표시합니다. `GEMINI.md` 파일을 변경한 경우 `/memory refresh`를 사용하여 세션을 다시 시작하지 않고 컨텍스트를 다시 로드할 수 있습니다.
*   **세션별 메모리**: 임시 컨텍스트의 경우 `/memory add` 명령어 다음에 현재 세션에서 AI가 기억하기를 원하는 텍스트를 추가할 수 있습니다.
*   **컨텍스트 압축**: 긴 대화에서 토큰을 절약하려면 `/compress` 명령어를 사용하세요. 이렇게 하면 현재 채팅 컨텍스트가 요약으로 대체됩니다.

## 파일 및 시스템과 상호 작용

*   **파일 또는 디렉토리 내용 주입**: `@` 기호 다음에 파일 또는 디렉토리 경로를 사용하여 프롬프트에 해당 내용을 포함시킵니다. 예를 들어, `@src/my_project/ 이 코드를 요약해 줘`. CLI는 git을 인식하며 `.gitignore`에 나열된 파일을 무시합니다.
*   **셸 명령어**: 느낌표(`!`)를 접두사로 사용하여 CLI에서 직접 셸 명령어를 실행할 수 있습니다. 예를 들어, `!ls -l`. 또한 `!`를 단독으로 입력하여 "셸 모드"를 토글할 수 있습니다.
*   **안전 제일**: 파일 시스템을 수정하는 명령어의 경우 CLI를 시작할 때 `--checkpointing` 플래그를 사용하는 것을 고려하세요. 이렇게 하면 파일 수정 전에 프로젝트의 스냅샷이 저장되며, `/restore`를 사용하여 변경 사항을 되돌릴 수 있습니다. `--yolo` 플래그는 모든 도구 호출을 자동 승인하는 데 사용할 수 있지만 주의해서 사용해야 합니다.

## 고급 기능

*   **채팅 관리**: 대화를 저장하고 다시 시작할 수 있습니다. `/chat save <tag>`를 사용하여 현재 세션을 저장하고 `/chat resume <tag>`를 사용하여 나중에 다시 불러올 수 있습니다. `/chat list`는 저장된 모든 대화를 표시합니다.
*   **Vim 모드**: 텍스트 입력에 Vim 키 바인딩을 선호하는 경우 `/vim` 명령어로 이 기능을 토글할 수 있습니다.
*   **IDE 통합**: Gemini CLI는 VS Code와 같은 IDE와 통합하여 더 원활한 워크플로우를 만들 수 있습니다.
